# Amnezia Split

Минималистичное расширение для Chrome и Brave: выбранные сайты идут через ваш сервер Amnezia, остальные — напрямую.

![Окно Amnezia Split](docs/popup.png)

**[Скачать готовое расширение (ZIP)](https://github.com/rub1kub/amnezia-split-extension/releases/latest)**

## Что умеет

- сразу включает ChatGPT, OpenAI, Sora и связанные домены;
- объединяет и ежедневно обновляет компактные списки от [itdoginfo/allow-domains](https://github.com/itdoginfo/allow-domains) и [1andrevich/Re-filter-lists](https://github.com/1andrevich/Re-filter-lists), включая отдельный список Google AI;
- показывает все домены через VPN с быстрым поиском и указанием источника;
- добавляет или исключает текущий сайт одним переключателем;
- хранит несколько серверов и переключает активный сервер прямо из компактного окна;
- не отправляет обычные сайты через VPN;
- хранит пароль прокси только в локальном хранилище браузера;
- показывает аккуратную плашку в окне расширения и у верхнего края страницы после установки новой версии или при доступном обновлении;
- при сбое прокси не переключает выбранный сайт на прямое соединение.

> Правила применяются по домену. Поэтому `chatgpt.com` будет идти через VPN во всех вкладках, а любой домен вне списка — напрямую. Один и тот же домен нельзя одновременно открыть через VPN в одной вкладке и напрямую в другой: API Chrome задаёт прокси на уровне профиля браузера.

## Установка в Brave с автообновлениями

Готовая CRX-версия устанавливается политикой Brave и автоматически получает новые версии из GitHub. Для текущего пользователя Windows запустите:

```powershell
powershell -ExecutionPolicy Bypass -File scripts/install-brave.ps1
```

Перезапустите Brave. Расширение появится в основном профиле и откроет страницу настройки. Введите пароль прокси и нажмите **Сохранить и проверить**.

Для разработки можно по-прежнему открыть `brave://extensions`, включить **Режим разработчика** и загрузить папку проекта как распакованное расширение. У такой установки автообновлений нет.

Адрес `ton4.pro`, порт `18443` и логин `amnezia-browser` уже заполнены. Пароль не хранится в репозитории. Другие серверы добавляются кнопкой **+ Добавить**; активный сервер выбирается в настройках или прямо в окне расширения.

## Как пользоваться

Нажмите значок расширения:

- большой переключатель ставит всю выборочную маршрутизацию на паузу;
- переключатель **Текущий сайт** отправляет открытый домен через VPN или добавляет его в исключения;
- **Список для России** включает готовый набор доменов.

Кнопка **Посмотреть домены** открывает полный список с поиском. Дополнительные сайты и исключения также видны в настройках расширения.

## Почему нужен отдельный прокси

Расширение браузера не умеет подключаться к туннелю Amnezia напрямую. На VPN-сервере развёрнут HTTPS CONNECT-прокси:

```text
Brave → TLS → ton4.pro:18443 → интернет
```

PAC-правила Chrome выбирают этот маршрут только для нужных доменов. TLS скрывает логин, пароль и запросы к прокси от провайдера. Подробности — в [документации сервера](docs/server.md).

## Разработка

Требуется Node.js 20 или новее.

```powershell
npm test
npm run package
npm run package:crx
```

Готовые ZIP и подписанный CRX появятся в `dist`. Закрытый ключ подписи хранится вне репозитория в `%LOCALAPPDATA%\Amnezia Split` и нужен для сохранения постоянного ID расширения при обновлениях.

## Приватность и разрешения

Расширение не собирает аналитику и не имеет внешнего бэкенда. Объяснение разрешений и работы с данными находится в [PRIVACY.md](PRIVACY.md).

Снимок `data/inside-raw.lst` получен из публичного списка [itdoginfo/allow-domains](https://github.com/itdoginfo/allow-domains/blob/main/Russia/inside-raw.lst) 15 июля 2026 года. При обновлении расширение объединяет его с компактным [community.lst](https://github.com/1andrevich/Re-filter-lists/blob/main/community.lst) и отдельным [списком Google AI](https://github.com/itdoginfo/allow-domains/blob/main/Services/google_ai.lst).
