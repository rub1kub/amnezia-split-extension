# Политика конфиденциальности Routeva

Дата вступления в силу: 15 июля 2026 года.

Routeva работает в Chrome/Brave и с сервером Routeva Gateway, который указал пользователь. Расширение не содержит аналитики, рекламы и облачного бэкенда разработчика. Разработчик не получает и не продаёт пользовательские данные.

## Данные на устройстве

В `chrome.storage.local` сохраняются:

- адреса, порты, типы протоколов, логины и пароли прокси;
- сведения о добавленных подписках и полученный от Gateway список узлов; персональная ссылка остаётся локально для управления и сохраняется также на выбранном пользователем Gateway для обновлений;
- пользовательские домены и исключения;
- локальная копия публичного списка доменов;
- определённые страна и публичный IP выхода;
- время обновлений, скрытые уведомления и последняя техническая ошибка прокси.

Эти данные не синхронизируются через аккаунт браузера и не передаются разработчику. Удаление расширения удаляет его локальное хранилище.

## Внешние запросы

Расширение обращается к:

- `raw.githubusercontent.com/itdoginfo/allow-domains` и `raw.githubusercontent.com/1andrevich/Re-filter-lists` — обновление публичных списков доменов;
- `raw.githubusercontent.com/rub1kub/amnezia-split-extension` — проверка номера доступной версии;
- `api.country.is` и резервный `ipwho.is` — определение страны и публичного IP текущего выхода при проверке или переключении сервера;
- Routeva Gateway, который настроил пользователь, — защищённая передача ссылки подписки, получение списка узлов и выбор активного выхода;
- выбранному пользователем прокси — передача трафика отмеченных доменов или всего интернета, когда пользователь явно выбрал соответствующий режим.

Gateway по указанию пользователя обращается к домену подписки для загрузки и обновления списка, а выбранный узел подписки передаёт интернет-трафик. HTTPS-ссылка может содержать секретный токен: она отправляется только на Gateway пользователя, хранится там с ограниченными правами и не попадает в GitHub или инфраструктуру разработчика.

## Разрешения

- `proxy` — применить PAC-правила выборочной маршрутизации;
- `webRequest` и `webRequestAuthProvider` — передать логин и пароль только запрашивающему их прокси;
- `storage` — сохранить локальные настройки;
- `tabs` — показать домен активной вкладки и изменить его маршрут;
- `alarms` — по расписанию обновить публичные списки и проверить версию;
- `<all_urls>` — применить выбранный режим прокси и авторизацию, обратиться к выбранному пользователем Gateway и показать локальную плашку версии. Расширение не читает и не сохраняет содержимое страниц.

## Ограниченное использование

Использование и передача данных соответствуют требованиям Chrome Web Store User Data Policy, включая Limited Use: данные используются только для заявленной функции выборочной маршрутизации, настройки прокси, импорта подписок и отображения состояния. Они не используются для рекламы, кредитной оценки, перепродажи или человеческого просмотра вне действий, необходимых для безопасности и поддержки по явному запросу пользователя.
