# Обоснование разрешений для Chrome Web Store

Тексты ниже можно вставить в раздел Privacy practices.

## Single purpose

Расширение выборочно направляет сетевой трафик доменов, выбранных пользователем или включённых в публичный список, через настроенный пользователем прокси.

## proxy

Нужно для установки PAC-правил, которые выбирают прямое соединение или настроенный прокси для каждого домена.

## webRequest

Нужно только для определения запроса аутентификации от настроенного прокси. Содержимое посещаемых страниц не читается и не сохраняется.

## webRequestAuthProvider

Нужно для безопасной передачи сохранённых локально логина и пароля прокси в ответ на его запрос авторизации.

## storage

Нужно для локального хранения серверов, подписок, списка доменов, исключений и пользовательских настроек.

## tabs

Нужно для отображения домена активной вкладки в popup и включения/исключения этого домена по действию пользователя.

## alarms

Нужно для периодического обновления публичных списков доменов и проверки доступной версии без постоянно работающего фонового процесса.

## Host permission: all URLs

Нужно, чтобы PAC-маршрутизация и прокси-аутентификация работали для всех ресурсов выбранного сайта, включая поддомены, API, изображения и WebSocket. Также локальный content script показывает только уведомление о версии; он не читает и не передаёт содержимое страниц.

## Optional host permission: HTTPS URLs

Запрашивается для одного конкретного origin только после того, как пользователь вставил HTTPS-ссылку подписки и нажал «Добавить подписку». Нужно для загрузки и последующего обновления этой подписки.

## Remote code

Удалённый код не используется. Все исполняемые JavaScript-файлы находятся внутри пакета. По сети загружаются только данные: текстовые списки доменов, JSON версии, ответ сервиса страны/IP и данные подписки, явно указанной пользователем. Они разбираются как данные и никогда не выполняются через `eval`, `Function` или удалённый script.
