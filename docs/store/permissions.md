# Обоснование разрешений для Chrome Web Store

Тексты ниже можно вставить в раздел Privacy practices.

## Single purpose

Расширение направляет выбранные домены или, по явному выбору пользователя, весь интернет через настроенный пользователем прокси; исключения остаются на прямом подключении.

## proxy

Нужно для установки PAC-правил, которые выбирают прямое соединение или настроенный прокси для каждого домена.

## webRequest

Нужно только для определения запроса аутентификации от настроенного прокси. Содержимое посещаемых страниц не читается и не сохраняется.

## webRequestAuthProvider

Нужно для безопасной передачи сохранённых локально логина и пароля прокси в ответ на его запрос авторизации.

## storage

Нужно для локального хранения серверов, подписок, списка доменов, исключений и пользовательских настроек.

## tabs

Нужно для отображения домена активной вкладки в popup и включения/исключения этого домена по действию пользователя.

## alarms

Нужно для периодического обновления публичных списков доменов и проверки доступной версии без постоянно работающего фонового процесса.

## Host permission: all URLs

Нужно, чтобы выбранный режим PAC-маршрутизации и прокси-аутентификация работали для сетевых ресурсов доменов или всего интернета. Разрешение также позволяет обратиться к Routeva Gateway, который пользователь явно настроил. Локальный content script показывает только уведомление о версии; он не читает и не передаёт содержимое страниц.

## Remote code

Удалённый код не используется. Все исполняемые JavaScript-файлы находятся внутри пакета. По сети загружаются только данные: текстовые списки доменов, JSON версии, ответ сервиса страны/IP и JSON выбранного пользователем Gateway. Они разбираются как данные и никогда не выполняются через `eval`, `Function` или удалённый script.
