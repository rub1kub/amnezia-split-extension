# Чек-лист публикации

## До загрузки

- [ ] `npm ci`
- [ ] `npm test`
- [ ] `npm run package:store`
- [ ] Проверить, что `manifest.json` лежит в корне ZIP.
- [ ] Проверить версию, описание и отсутствие `update_url`.
- [ ] Установить папку `dist/routeva-store` как распакованное расширение в чистый профиль.
- [ ] Проверить popup, настройки, импорт тестовой подписки, HTTP/HTTPS/SOCKS и удаление локальных данных.
- [ ] Убедиться, что в архиве нет реальных паролей, токенов и пользовательских ссылок.

## В Chrome Web Store Developer Dashboard

- [ ] Войти в Google-аккаунт разработчика и оплатить разовый регистрационный сбор, если ещё не оплачен.
- [ ] Создать новый item и загрузить ZIP `dist/routeva-<версия>-store.zip`.
- [ ] Вставить тексты из `listing-ru.md`.
- [ ] Загрузить иконку 128×128 и минимум один подготовленный скриншот 1280×800.
- [ ] Заполнить обоснования разрешений из `permissions.md`.
- [ ] Заполнить Data usage по `privacy-fields.md` и указать публичную политику конфиденциальности.
- [ ] Выбрать распространение и отправить на проверку.

## После публикации

- [ ] Установить магазинную версию в отдельный профиль и проверить автоматическое обновление.
- [ ] Добавить ссылку Chrome Web Store в README и GitHub Release.
- [ ] Для следующих релизов увеличить версию и загрузить новый ZIP в тот же item.

Публикация не может быть завершена без входа владельца в Google Developer Dashboard и, при необходимости, оплаты регистрационного сбора. Код и архив не требуют CRX или системной политики.
